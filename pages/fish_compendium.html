Fish Compendium

Project

This project was completed as part of an assignment on creating a "filterable, sortable, searchable" catalogue interface.

A live version of this project can be found here.

Goals

The "Fish Compendium" provides a modern, simple catalogue for the fish collectables in the video game Animal Crossing: New Leaf.

It aims to create an interface for players to easily keep track of which fish species they have collected, look up how to find the species that they are missing, and filter or search for certain species. A player can set filter settings to the current location and time of their game, and easily see which not-yet-caught fish are available to obtain; or simply search for certain fish to look up the times and locations where they are obtainable.

Design Choices

The interface of this project was designed with intuitive use and simplicity in mind.

The catalogue main page.
An "information card" for a specific species.

To create a consistent visual hierarchy, information is grouped in rounded boxes for easier parsing and reading. Additionally, clickable tags outside of the filter bar (e.g., the location tags on each species's information card) are styled with an accent color so that the user can easily understand which elements are interactable.

Options for filtering, sorting, and searching are clearly presented in a section at the top of the screen, and the catalogue's results are presented in graphic form reminiscent of information cards. Cards can also be clicked on to open a larger, more detailed window, which provides more context for a first-time user or user who is less experienced with Animal Crossing. All filter and tag buttons provide tooltip-like hints when hovered over, to create an invisible guide for new users.

User navigation is streamlined by the use of large, easily clickable buttons and clickable regions for filtering by tag or opening detailed information panes. Users can also click outside of detailed information panes to close them, as may be expected of modern applications.

To create a more comfortable user experience, all button and toggle actions are accompanied by a visual response (such as clicking on a "time found" filter tag, which changes the button's color to indicate that this tag is active). This choice reassures users that actions are being executed successfully, even if the effect of the selection on the catalogue is not apparent at first (e.g., choosing to filter only fish that appear during the day yields little change in the catalogue, since most fish appear in the day, but the highlighted filter button indicates that the filter was successfully applied).

Conclusion

Moving forward, I would love to add additional features to this project such as an export/import tool to save or load a player's own caught status for each fish, and additional tags and filter options for other, game-specific events.
